Customize where APIGate's built-in features are accessible. This allows you to serve your own content at standard paths (e.g., custom portal at /portal/) while accessing APIGate features at different paths (e.g., /apigate-portal/).

Settings Overview

View your APIGate configuration. These settings are read-only and configured via environment variables.

Upstream API

  • URL - Default backend API server. Can be overridden per-route.
  • Timeout - Maximum time to wait for upstream response.

Authentication

  • Mode - local (built-in) or remote (external service)
  • Header Name - HTTP header where API key is expected

Environment Variables

  • APIGATE_UPSTREAM_URL
  • APIGATE_UPSTREAM_TIMEOUT
  • APIGATE_AUTH_MODE
  • APIGATE_AUTH_HEADER
  • DATABASE_DSN

Configuration Reference

Complete list of environment variables for APIGate.

Upstream Settings

  • APIGATE_UPSTREAM_URL - Default backend URL
  • APIGATE_UPSTREAM_TIMEOUT - Request timeout (e.g., "30s")

Authentication

  • APIGATE_AUTH_MODE - "local" or "remote"
  • APIGATE_AUTH_HEADER - Header name for API key
  • APIGATE_JWT_SECRET - Secret for admin JWT tokens

Database

  • DATABASE_DSN - Connection string

Examples:

sqlite://data.db
postgres://user:pass@host/db

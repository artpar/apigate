How Routes Work

Routes define how incoming API requests are processed:

  1. Match - Request path and method are checked against route patterns
  2. Transform - Headers, query params, or body can be modified
  3. Forward - Request is sent to the target upstream
  4. Meter - Usage is extracted from the response for billing
Example Flow
Client: POST /v1/chat/completions ↓ Route: /v1/* → OpenAI upstream ↓ Add: Authorization header ↓ Forward to: https://api.openai.com/v1/chat/completions ↓ Extract: usage.total_tokens for metering Response: 200 OK (14 tokens used)

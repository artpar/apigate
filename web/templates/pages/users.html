Users

Users are your API consumers. Each user can have one or more API keys to access your gateway.

User Properties

  • Email - Unique identifier for the user
  • Plan - Determines rate limits and quotas
  • Status - Active or suspended

Managing Users

  • Create - Add a new API consumer
  • Edit - Change plan or status
  • Delete - Remove user (also revokes their keys)

User Plans

Each user is assigned a plan that controls their rate limits and monthly quota. You can change a user's plan at any time from the edit page.

Users API

Manage users programmatically.

Create User

POST /mod/user
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "securepassword",
  "plan_id": "free"
}

List Users

GET /mod/user

Update User

PUT /mod/user/{user_id}
Content-Type: application/json

{
  "plan_id": "pro",
  "status": "active"
}

Delete User

DELETE /mod/user/{user_id}
